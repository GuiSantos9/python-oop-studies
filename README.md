🏗️ Python Orientado a Objetos (POO)
Este repositório é dedicado ao estudo dos pilares e conceitos avançados de Programação Orientada a Objetos em Python. O objetivo aqui é aprender a pensar em termos de "objetos" e "classes" para construir sistemas mais robustos e escaláveis.

🚀 Progresso de Estudos - POO
Abaixo, os conceitos fundamentais que estou explorando. Cada item marcado representa um conceito compreendido e aplicado em código.

Fundamentos de Classes

[✅] Classes e Objetos - A planta (blueprint) vs. a instância real.

[✅] Método __init__ - O construtor da classe e inicialização de estados.

[✅] Parâmetro self - Entendendo a referência à própria instância.

[✅] Propriedades da Classe - Atributos de instância vs. atributos de classe.

[✅] Métodos da Classe - Comportamentos e ações que um objeto pode realizar.

Os Pilares da POO

[ ] Herança - Criando subclasses e reutilizando código de classes pai.

[ ] Polimorfismo - A capacidade de diferentes classes responderem à mesma mensagem de formas distintas.

[ ] Encapsulamento - Protegendo dados através de modificadores de acesso (público, protegido, privado).

Conceitos Avançados

[ ] Inner Classes - Criando classes dentro de outras classes para melhor organização lógica.

📖 Resumo do Repositório
Diferente da programação procedural, este módulo foca em:

Modelagem: Como traduzir entidades do mundo real para o código.

Abstração: Esconder detalhes complexos e mostrar apenas o necessário.

Manutenibilidade: Criar códigos que são fáceis de alterar sem quebrar o sistema inteiro.
